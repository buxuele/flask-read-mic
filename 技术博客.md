# 从草台班子到工业级：我的“西瓜红”实时转录引擎进化史

谁说搞 AI 工具非得整一堆花里胡哨的渐变色和点不完的弹窗？在这个项目里，我们只信奉一个教条：**极致的克制，就是最高的效率。** 

以下是我们将一个简陋的 Demo 调教成生产力怪兽的过程，全是干货，建议收藏。

### 一、 审美觉醒：向“科技蓝”和“全家桶”说不
最开始，UI 充满了廉价感，后来我们决定给它做一场彻头彻尾的“去油手术”。

1.  **视觉锚点**：废弃了所有细碎的小图标，首页只留一个 **3 倍大的“西瓜红” (#d9234a) 麦克风**。要录音？点它就行，别废话。
2.  **空间留白**：坚决干掉多层嵌套的“盒中盒”设计。顶部 Navbar 高度直接减半，把视野留给最值钱的文字。
3.  **致敬经典**：引入 ChatGPT 同款侧边栏图标。没有文字说明，因为那是全人类的审美共识，图标够经典，就不需要象形游戏。
4.  **反油腻交互**：删掉了所有的 `hover`（悬停）动效。按钮就是按钮，不需要它在鼠标移过去时给你表演发光。
5.  **原地反馈**：点“复制全部”，按钮原地变身“已复制!”。拒绝弹窗，拒绝阻断思考。

### 二、 核心黑科技：滑动窗口与“自我修正”逻辑
传统的转录是一段一段往外蹦，断句随缘，错别字满天。我们搞了一套“内功”：

1.  **句子级缓冲区**：我们在内存里开辟了一个 `SESSION_STATE` 秘密队列，录音不再是孤立碎片，而是会智能累加。
2.  **滑动纠错手术**：Whisper 每次拿到的都是过去几秒的音频全集。这意味着，当你还没说完后半句，系统已经根据语境帮你悄悄把前半句的错字改了。
3.  **封板机制**：一旦 Whisper 吐出“。！？”，这行字就立刻“封板存档”。下个字进来，是从灵开始的新的一页。
4.  **全自动清空**：只要点击录音，主屏立即“清场”，旧内容自动入库侧边栏。拒绝在一个屏幕里看两段不想干的信息。

### 三、 避坑指南：那些藏在代码里的冷知识
为了让它在 Windows 上稳如老狗，我们做了这些“脏活儿”：

1.  **CUDA 自动导航**：写了一套路径探测脚本，自动在 site-packages 里翻找 NVIDIA 的 DLL，解决了“有显卡却跑不起来”的世纪难题。
2.  **标点符号“猛药”**：Whisper 对短句很冷淡？我们通过 `initial_prompt` 注入了一段充满“暗示”的强力诱导，让它产出的标点比公文还规范。
3.  **VAD 宽限期**：把静音检测阈值拉到了 1000ms。哪怕你说话语速慢、爱喘气，系统也会耐心等你把句号说完。
4.  **死锁 localStorage**：将所有的偏好设置（模型、语种、选项）锁死在浏览器缓存里。一旦调好，终身受用。

### 四、 最后的总结
这个项目告诉我们：一个好工具，**不是功能堆叠出来的，而是把没用的地方砍掉剩下的精华。** 

现在的它，左边是“正文前 20 字”生成的智能索引，右边是极细、近乎隐形的优雅滑杆，中间是由于你的思考而不断律动的西瓜红。这就够了。
