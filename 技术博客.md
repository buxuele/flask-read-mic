# python + fast-wahisper 读取麦克风，实现语音转录，而且是实时转录。

### 项目地址 [github 链接](https://github.com/buxuele/flask-read-mic)



在将一个基于 Flask 的语音转录项目从“原型”转变为“专业工具”的过程中，我们不仅在做加法（核心算法），更在做大量的减法（视觉与逻辑冗余）。以下是本次重构的技术细节与审美决策。

### 1. 核心转录逻辑：从“碎分片”到“滑动窗口纠错”
传统的实时转录往往将音频切成固定时长的小块，容易导致断句错误。我们引入了全新的**句子级动态生命周期管理**：

-   **累加式推理**：后端通过 `SESSION_STATE` 机制，在用户未完成一个完整句子（未检测到止符）时，会将音频持续累加。Whisper 每次推理都能获得更充足的语境。
-   **外科手术式更正**：引入 `replace_start_index` 协议。当前端收到新结果时，如果属于同一句的延伸，它会精准替换之前的临时文本，实现类似“流式自动纠错”的效果。
-   **推理内核优化**：
    -   **Prompt 强诱导**：通过 `initial_prompt` 强制要求模型输出丰富的标点符号。
    -   **VAD 宽限**：将静音检测时长从 300ms 放宽至 1000ms，有效保护了慢语速下的长句完整性。

### 2. 审美宪法：一场彻底的“视觉大扫除”
为了追求极致的排版与克制感，我们执行了以下**强制删除**清单：

-   **删除了所有 Hover 动画**：这被视为低级交互。按钮状态应是静默且确定的。
-   **删除了“科技蓝”渐变**：严禁任何俗套的渐变色和外发光，统一采用纯正暗色背景。
-   **删除了占位废话**：例如“暂无转录内容”等无关痛痒的提醒，维持界面的极致留白。
-   **删除了松散按钮**：合并了零碎的“清空”、“删除”等操作，将视野留给转录本身。

### 3. 完成的核心功能清单
我们通过以下有序改进，确保了项目的工业级体验：

1.  **视觉锚点确立**：打造了 3 倍大小的西瓜红 (#d9234a) 核心录音按钮。
2.  **设置静默持久化**：利用 `localStorage` 自动记忆用户的识别模型、语言、时长偏好，消除重复操作。
3.  **智能索引系统**：侧边栏放弃了无意义的序号，自动提取转录全文前 20 字作为标题。
4.  **单会话聚焦**：点击录制即自动清除主屏旧内容并入库历史，确保 UI 逻辑符合物理直觉。
5.  **工程化加固**：
    -   自动探测 Windows 下的 CUDA DLL 路径。
    -   实现 favicon 规范化集成（static 目录迁移）。
    -   加固 `.gitignore`，物理隔离用户所有的个人音频与数据库隐私。

### 4. 总结
一个好工具的产出，其本质是**对人类直觉的尊重**。我们砍掉了那些干扰思考的视觉杂讯，只留下最纯粹的录制体验。当你点击那颗西瓜红按钮时，技术应当是隐形的。
