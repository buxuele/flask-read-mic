# UI 与审美要求总纲 (最新修订)

任何 UI 的增、删、改必须严格遵守以下“原始语录”所确立的审美纪律：

### 1. 核心审美哲学
- “更人性化的设计，更符合人类直觉的设计。”
- “绝佳的克制感：强烈偏好纯正的暗色主题。追求极致简洁配色。”
- “我喜欢更简洁的 ui, 更大方，不要多层嵌套（盒中盒）。”
- “多采用带有空间留白和阻尼感阴影的高级悬浮面板，版面必须要舒展、透气。”
- “极度厌恶并严格禁止带有‘科技蓝’的俗套渐变色，认为那样极为‘庸俗’。”
- “彻底删除所有的鼠标悬停 (hover) 动画，这非常低级。”
- “绝！对！不！要！写 emoji！”

### 2. 布局与组件纪律
- “导航栏保证易用性与统一性，高度要减半紧凑，不要留下大片无意义的垂直空白。”
- “录音按钮更大一些。3倍！而且，把红色改为西瓜红色 #d9234a，作为大方的视觉核心。”
- “侧边栏折叠等极经典的通用共识位，必须用经典图标(如 ChatGPT 式无字 svg)，不要塞笨重的文字。”
- “除极少数通用图标外，按键必须带文字标签，不能仅有图标让人去猜。”

### 3. 内容展示与交互（最新追加）
- **单会话聚焦**：“为啥这里会显示多个翻译内容区域？我想只保留一个。点击录音按钮，自动清空当前元素并加入历史。”
- **文本即标题**：“侧边栏标题要把翻译文本的前 20 个字作为标题，日期时间作为副标题。”
- **拒绝废话占位**：“‘暂无转录内容’ 这一行字删掉！不需要任何画蛇添足的提醒。”
- **极简滑杆**：“如果历史记录很长，右边的滑杆必须极细、极隐蔽，绝不破坏整体美感。”
- **原地反馈**：“点击复制全部按钮，应该在按钮上原地显示‘已复制!’，提供更友好的直觉闭环。”
- **异常透明**：“尽量少写 try 语句包揽全局，直接让真实的系统报错信息完整打印出来（不写大块注释）。”
