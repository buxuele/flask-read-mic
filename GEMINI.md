
### 1. 核心纪律：严禁“局部缝合”，执行先会话后干活机制
- **绝对拒绝挤牙膏**：每次接手 UI 或代码需求，动手前必须进行**全局审查 (Global Audit)**。举一反三，将同类的庸俗样式或冗余代码一次性彻底清理完，绝不要打补丁的缝合怪。
- **强制沟通（先分析后动手）**：遇到代码修改或 Bug 修复时，严禁毫无沟通直接“埋头拉车”。必须先在回复中清晰地向我阐明你的思路和诊断结果。交流完原因之后，紧接着再调用工具干活。一定要体现出有脑子的结对编程人性化底线。
- **强制自检**：交差前必须在心里自检 3 件事：是否违背了审美死规矩？是否引发了级联破坏？是否彻底删掉了该删的冗余代码？如果还要我充当测试员帮你找漏，即视为严重失职。

### 2. 高级审美哲学：克制、舒展、直觉优先
- **色彩与质感**：强制保持纯正的暗配色，追求极致的简洁大方。极度厌恶“科技蓝”和任何俗套的外发光或渐变色，认为那极其庸俗。禁止 Emoji。
- **空间与层次（去嵌套化）**：严禁导致局促感的“多层嵌套（盒中盒）”设计。多采用带有空间留白和阻尼感阴影的“高级悬浮面板”，版面必须要舒展、透气。
- **反油腻交互**：彻底删除所有的“鼠标悬停（Hover）动效”。交互逻辑必须符合人类本能直觉（不需要的防呆警告坚决干掉，系统行为不得违背物理常识）。
- **极简且能懂**：导航栏要高度统一紧凑。所有的可操作按键**必须毫不例外地带有中文文字标签**，绝不接受“反人类”的独立象形图标按钮。

### 3. 底层代码观：追求绝对纯粹与极度透明
- **不要藏着掖着**：极度抵触 `try...except` 的兜底滥用。宁可让程序完整抛出红色报错异常暴露在控制台上，也绝不要吃掉错误并导致系统产生暗病和卡顿。
- **全透明配置管理**：排斥复杂的命令行传参交互。将所有核心参数和开关明明白白地定义成全局变量，直接丢在代码文件的最顶端。
- **大扫除式重构**：只要遇到历史遗留、引起冲突的老旧逻辑，绝不绕道走缝合路线，必须当场大刀阔斧地重写碾碎。

### 4. 项目外围运行卫生与知识沉淀
- **沉默的知识金库**：必须自动维护并经常更新以下文件，不要长篇废话：
    1. `项目记录.md`：极简记录项目整体状态变更。
    2. `过程记录.md`：原字原句地摘录我的每一次要求。
    3. `提示词记录.md`：持续补充提取的通用规则与经验避坑机制。每次完成任务自动执行这三项的同步。
- **环境安全卫生**：极其注意 Windows 系统的反斜杠陷阱；启动 Python 项目绝对要求绑定 venv 虚拟环境并且有更新 requirements.txt 的强迫症；遭遇网络瓶颈主动秒切国内源。
- **生命周期责任制**：严禁为了走捷径滥发大范围的 `taskkill` 洗地。凡是你调用的终端、启动的后台进程，由你负责精准定位并干净利落地关闭。
