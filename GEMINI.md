### 核心工作纪律：严禁“挤牙膏”与“局部打补丁”
- **全局审查 (Global Audit)**：每次接手代码或 UI 需求时，绝对别只改我指出的那一行。动手前必须扫描全文件或相关组件，举一反三，把所有同类问题（比如俗套的样式、冗余的遗留代码）一次性彻底清理完。
- **强制自检**：交差前必须在心里自检：
    1. 违反我的“死规矩”（如emoji、hover动画）了吗？
    2. 这个修改会导致级联破坏被撑爆吗？
    3. 对于冗余、老旧的代码，是否果断大刀阔斧地删除了？（我不要打补丁的缝合怪）。
- 如果让我来当你的人肉测试员，即视为你的极度失职。

### 沟通第一：前置你的大脑
- **谋定而后动**：不管写代码还是改代码，动手之前必须先讲一下你的思路：你要改什么？影响面是什么？把分析结果发送显示出来，然后再使用工具写代码，拒绝长时间黑盒作业。
- **长耗时反馈**：只要涉及长耗时任务，必须提供明确的进度反馈。

### 编程与代码规范
- **死规矩**：绝！对！不！要！写 emoji！
- **精简与透明**：尽量少写 try 语句包揽全局，直接让真实的系统报错信息完整打印出来。不写大块代码注释；文档说明必须高度浓缩（能写两句绝不写五句）。
- **Python 专属约束**：
    1. 新建 Python 项目必建 venv 虚拟环境。经常检查并更新 requirements.txt 与 readme。
    2. 遇到包报错，去 venv 里翻具体的源码，看它是怎么写的。如果下载包过慢，主动换算力镜像。
    3. 不要写命令行参数。把关键参数全部改写成全局变量，放在代码文件顶部。
- 代码写完后，必须二次自检排查是否存在语法错误并当场处理。对于底层架构、API平台或全局变量的改动，严禁只改配置文件，必须**先进行全局搜索**评估影响。

### 我的高级 UI 审美（专业与克制）
- **绝佳的克制感**：强烈偏好纯正的暗色主题。追求极致简洁配色，**极度厌恶并严格禁止带有“科技蓝”的俗套渐变色**，认为那样极为“庸俗”。彻底删除所有的“鼠标悬停 (hover) 动画”，这非常低级。
- **布局与易用性**：组件大量运用“悬浮面板”以构造空间层级。导航栏保证易用性与统一性，不接受松散零乱的按钮布局。
- **直觉操作**：按键**必须全部带有文字标签**，不能仅有图标让人去猜。操作要求极致的“直观”与“简易”。

### 环境与外围清理
- 操作文件时必须注意 Window 平台的斜杠方向限制。如果是创建新文件夹，务必先检查存在与否。
- 终端环境中，bash 里面严禁使用反斜杠换行拼接。
- 请机灵点儿：自动使用各项 Skills, MCP, Playwright 等挑选最合适的干活工具。
- **进程管理**：严禁随意使用 `taskkill` 因为你无法确定哪个是我自己机器上的同名进程；你的纪律是：凡是**由你亲手开启的**终端和挂起的进程，用完之后必须由你主动、精确地全部安全关闭。
